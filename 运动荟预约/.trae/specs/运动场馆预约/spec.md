# 运动场馆预约软件 - 产品需求文档

## Overview
- **Summary**: 使用uni-app框架开发一个运动场馆预约应用，包含3个TabBar页面：首页（场馆分类入口+搜索）、场馆列表、我的预约。支持H5和微信小程序双端运行，数据使用静态数据。
- **Purpose**: 为用户提供一个便捷的运动场馆预约浏览入口，可以按场馆类型筛选、按名称搜索、查看预约记录。
- **Target Users**: 需要预约运动场馆的普通用户

## Goals
- 实现3个TabBar页面（首页、场馆列表、我的预约）
- 实现场馆分类导航（羽毛球馆、篮球馆、乒乓球馆等），点击跳转对应类型的场馆列表
- 实现搜索功能，根据输入的场馆名称直接跳转到场馆详情
- 实现场馆列表页，展示场馆卡片（名称、金额、距离）
- 实现我的预约列表及详情页（场馆名称、地址、预约时间）
- 支持H5和微信小程序双端运行
- 使用静态数据，无后端API依赖

## Non-Goals (Out of Scope)
- 不实现支付功能
- 不实现登录/注册功能（用户身份使用模拟数据）
- 不实现预约提交/取消功能
- 不实现后端API接口
- 不实现地图导航功能
- 不实现收藏/分享功能

## Background & Context
- 这是一个从0到1的uni-app新项目
- 技术栈：uni-app + Vue3（或Vue2，根据uni-app默认模板）
- 需要支持H5和微信小程序双端运行
- 数据全部使用本地静态JSON数据

## Functional Requirements
- **FR-1**: TabBar导航 - 底部有3个TabBar：首页、场馆列表、我的预约
- **FR-2**: 首页-场馆分类 - 展示羽毛球馆、篮球馆、乒乓球馆等分类入口，点击跳转到对应类型的场馆列表
- **FR-3**: 首页-搜索功能 - 顶部搜索框，输入场馆名称后直接跳转到对应场馆的详情页
- **FR-4**: 场馆列表页 - 以卡片形式展示所有场馆，每个卡片包含：场馆名称、金额、距离
- **FR-5**: 场馆详情页 - 点击场馆卡片进入详情页，展示场馆名称、地址、简介、可用时段等信息（可预约）
- **FR-6**: 我的预约列表 - 展示当前用户已预约的场馆列表
- **FR-7**: 预约详情页 - 展示预约的场馆名称、场馆地址、预约时间

## Non-Functional Requirements
- **NFR-1**: 跨平台兼容 - 代码可同时在H5和微信小程序环境运行
- **NFR-2**: 响应式UI - 适配常见手机屏幕尺寸
- **NFR-3**: 页面跳转流畅 - 页面切换无明显卡顿
- **NFR-4**: 代码可维护性 - 静态数据集中管理，组件结构清晰

## Constraints
- **Technical**: 
  - 必须使用uni-app框架
  - 只能使用静态数据，不能调用任何后端API
  - 支持H5和微信小程序
- **Business**: 无特殊业务约束
- **Dependencies**: 依赖uni-app官方模板及基础组件

## Assumptions
- 用户身份固定（模拟单用户场景），无需登录
- 预约记录是预定义的静态数据
- 场馆详情页的"预约"按钮仅做跳转演示或弹窗提示，不实际提交
- 搜索功能基于完全匹配或包含匹配的静态数据搜索

## Acceptance Criteria

### AC-1: TabBar导航显示
- **Given**: 应用已启动
- **When**: 用户查看底部导航
- **Then**: 显示3个TabBar：首页、场馆列表、我的预约
- **Verification**: `human-judgment`
- **Notes**: 检查TabBar图标和文字是否正确

### AC-2: 首页-场馆分类入口跳转
- **Given**: 用户在首页
- **When**: 点击"篮球馆"分类入口
- **Then**: 跳转到场馆列表页，且列表只显示篮球馆类型的场馆
- **Verification**: `programmatic` (检查路由参数和过滤结果) + `human-judgment`

### AC-3: 首页-搜索功能跳转详情
- **Given**: 用户在首页搜索框
- **When**: 输入已存在的场馆名称（如"阳光篮球馆"）并搜索
- **Then**: 直接跳转到该场馆的详情页，信息匹配
- **Verification**: `programmatic`
- **Notes**: 如果搜索不到，给予友好提示

### AC-4: 场馆列表页卡片展示
- **Given**: 用户在场馆列表页
- **When**: 查看任意场馆卡片
- **Then**: 每个卡片显示：场馆名称、金额（价格）、距离
- **Verification**: `human-judgment`

### AC-5: 场馆列表点击跳转详情
- **Given**: 用户在场馆列表页
- **When**: 点击任意场馆卡片
- **Then**: 跳转到该场馆的详情页
- **Verification**: `programmatic`

### AC-6: 我的预约列表展示
- **Given**: 用户在"我的预约"Tab
- **When**: 查看预约列表
- **Then**: 展示该用户已预约的场馆列表（至少1条数据）
- **Verification**: `human-judgment`

### AC-7: 预约详情页信息
- **Given**: 用户在我的预约列表
- **When**: 点击某条预约记录
- **Then**: 跳转到预约详情页，展示：场馆名称、场馆地址、预约时间
- **Verification**: `human-judgment` + `programmatic`

### AC-8: H5环境可运行
- **Given**: 代码已开发完成
- **When**: 运行 `npm run dev:h5` 并在浏览器打开
- **Then**: 页面正常显示，TabBar、跳转功能正常工作
- **Verification**: `programmatic` (检查构建成功) + `human-judgment`

### AC-9: 微信小程序环境可运行
- **Given**: 代码已开发完成
- **When**: 运行 `npm run dev:mp-weixin` 并在微信开发者工具导入
- **Then**: 页面正常显示，TabBar、跳转功能正常工作
- **Verification**: `human-judgment` (需要开发者工具验证)
- **Notes**: 本项目验收以H5端自测通过为准，小程序端确保代码结构兼容
